AI Facilitation for Academic Discussions

An AI-based facilitation service for asynchronous academic discussions: it reads the conversation, decides whether a facilitation message would add value, and generates one if so.

How it works

  1. Classify Reads the full thread and scores the conversation across several dimensions: engagement level, participation balance, topical depth, discussion phase, and discourse quality.
  2. Decide Weighs the classification scores and determines whether stepping in would help the discussion or interrupt it. Most threads get no intervention.
  3. Facilitate If an intervention is warranted, selects a facilitation role (organizational, intellectual, social, affective, or moderator) and generates a message using a targeted technique from the repertoire — a question, a summary prompt, a rebalancing move, and so on.

Both environments run the same pipeline. IDRIL uses local LLMs on UCM hardware via Ollama; MGMDY uses external LLMs via OpenRouter.

Try it Open the dashboard, go to Single thread, paste a discussion thread, and submit. The output shows the classification scores, whether the system decided to intervene, and the facilitation message if one was produced.
