Diseño, implementación e integración de un modelo de moderación inteligente para discusiones académicas en plataformas de aprendizaje abiertas

An AI-based facilitation service for asynchronous academic discussions: it classifies conversation state, decides whether to intervene, and generates a facilitation message.

María Grimaldi · UCM, Máster en Ingeniería Informática · 2026 · GitHub

Environments

IDRIL

idril.fdi.ucm.es/2526-moderacion

UCM academic server. Runs the facilitation pipeline with local Ollama models on UCM hardware. Requires UCM network or VPN.

ministral-3:14b (local, Ollama)

MGMDY

facilitation.mgmdy.xyz

AWS EC2 deployment. Runs the same pipeline with external models via OpenRouter.

gpt-4o (OpenRouter)

How it works

  1. Classify Reads the thread and scores the conversation state: engagement level, participation balance, topical depth.
  2. Decide Determines whether an intervention adds value at this point.
  3. Facilitate If yes, generates a facilitation message appropriate to the conversation state.

Both environments run the same pipeline. IDRIL uses local LLMs on UCM hardware via Ollama; MGMDY uses external LLMs via OpenRouter.

Try it Open the dashboard, paste a discussion thread, and submit. The output shows the classification scores, the intervention decision, and the generated message if one was produced.
