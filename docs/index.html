An AI-based facilitation service for asynchronous academic discussions: it classifies conversation state, decides whether to intervene, and generates a facilitation message.

gpt-4o (OpenRouter)

How it works

The facilitation service receives an asynchronous discussion thread and runs a three-stage pipeline: it classifies the conversation state (engagement level, participation balance, topical depth), decides whether an intervention adds value, and generates a facilitation message if so.

The two environments run the same codebase. IDRIL uses local LLMs on UCM hardware via Ollama. MGMDY uses external LLMs via OpenRouter API.

To try it: open the dashboard, paste a discussion thread, and submit. The output shows classification scores, the intervention decision, and the generated message if one was produced.
